# Pre-application checklist

Everything that needs to be true before contacting Garmin Developer
Relations. Order is roughly the order to execute in — but several can
run in parallel.

| # | Item | Owner | Blocks | Status |
|---|---|---|---|---|
| 0 | Pick brand name; confirm `.com` / `.io` / `.run` availability on Namecheap | you | everything below | ☐ |
| 1 | Buy domain | you | landing page, email, application | ☐ |
| 2 | Incorporate (LLC / Ltd / BV / similar) | you | privacy / terms / Garmin app | ☐ |
| 3 | Set up `peter@switchback.run` (Google Workspace or Fastmail) | you | Garmin app submission | ☐ |
| 4 | Publish landing page (Framer / Webflow / Carrd / Next.js — pick one) | you | Garmin app references it | ☐ |
| 5 | Privacy policy + Terms reviewed by lawyer, published at `/privacy`, `/terms` | you + lawyer | Garmin app | ☐ |
| 6 | Product brief finalized (`brief.md`) | you | Garmin app attachment | ☐ |
| 7 | Garmin application copy finalized (`garmin-application.md`) | you | submission | ☐ |
| 8 | Submit via Garmin Developer contact form (`garmin-contact-ticket.md`) | you | response from Garmin | ☐ |

## Notes on dependencies

- **Brand decision (#0) blocks 7 of 8 items.** Don't start on landing page
  copy customization, domain purchase, or email setup until the name
  is locked in.
- **Incorporation (#2) blocks 4.** Privacy policy and terms need an
  entity name; Garmin's application asks who's applying.
- **Lawyer review (#5) is the longest-lead item** outside Garmin's
  timeline. Find one this week, get a quote, start.
- **Items 4, 6, 7 can be drafted in parallel** as soon as the brand is
  picked.

## Realistic timeline

| Week | Goal |
|---|---|
| 1 | Brand + domain locked. Incorporation paperwork filed. Lawyer engaged. |
| 2 | Email live. Landing page draft published. Privacy/terms in lawyer's hands. |
| 3 | Privacy/terms live. Brief finalized. Application drafted. |
| 4 | Application submitted via contact form. Begin building MVP on `.FIT` export in parallel while waiting. |

After week 4, the critical path is Garmin's response. Use the wait
time to ship the `.FIT`-export MVP and get to 20–50 alpha users —
"we have demand" is leverage when Garmin replies.
